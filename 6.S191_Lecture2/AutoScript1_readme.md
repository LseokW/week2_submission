# YT Script Extractor — 기술 문서

## 사용 라이브러리

| 라이브러리 | 역할 |
|---|---|
| `yt-dlp` | 유튜브 자막 데이터 추출 |
| `streamlit` | 웹 UI 구성 |
| `tempfile`, `json`, `re` | 파일 처리 및 텍스트 파싱 (Python 표준 라이브러리) |

---

## yt-dlp 개요

`yt-dlp`는 유튜브를 비롯한 다양한 스트리밍 플랫폼에서 영상 및 메타데이터를 추출할 수 있는 오픈소스 라이브러리입니다. 본래 영상 다운로드 목적으로 설계되었으나, `skip_download` 옵션을 활성화하면 영상 본체를 다운로드하지 않고 자막 데이터만 선택적으로 수집할 수 있습니다.

유튜브는 자막을 두 가지 방식으로 제공합니다.

- **수동 자막**: 콘텐츠 제작자가 직접 작성한 자막으로, 구두점 및 줄바꿈이 포함되어 있습니다.
- **자동 생성 자막**: 유튜브의 음성 인식 AI가 자동으로 생성한 자막으로, 구두점 및 문장 경계 정보가 포함되지 않습니다.

본 애플리케이션은 두 가지 자막을 모두 시도하며, 존재하는 자막을 우선적으로 가져옵니다.

---

## 자막 포맷

`yt-dlp`를 통해 수집 가능한 자막 포맷 중 본 애플리케이션은 아래 두 가지를 순서대로 시도합니다.

### 1. json3 포맷 (우선 시도)

유튜브 내부 포맷으로, 타임스탬프가 밀리초(ms) 단위로 정밀하게 기록됩니다.

```json
{
  "events": [
    {
      "tStartMs": 1200,
      "segs": [{ "utf8": "Hello world" }]
    }
  ]
}
```

`tStartMs` 값을 `00:01`과 같은 시각 형식으로 변환하여 사용합니다.

### 2. VTT 포맷 (대체 수단)

WebVTT 표준 포맷으로, json3 자막이 존재하지 않을 경우 사용됩니다.

```
00:00:01.200 --> 00:00:03.400
Hello world
```

`-->` 구분자를 기준으로 타임스탬프와 텍스트를 분리하여 파싱합니다.

## 처리 흐름

```
유튜브 URL 입력
      ↓
yt-dlp를 통해 자막 파일 다운로드 (임시 디렉터리 사용)
      ↓
json3 파일 존재 시 → parse_json3() 호출
존재하지 않을 시 vtt 포맷으로 재시도 → parse_vtt() 호출
      ↓
세그먼트 리스트로 정리
[{ "start": "00:01", "text": "Hello world" }, ...]
      ↓
사용자 설정(타임스탬프 포함 여부, 줄 합치기 여부) 적용
      ↓
화면 출력 및 txt 파일 다운로드 제공
```

---

## 한계 및 고려 사항

- **구두점 부재**: 자동 생성 자막은 유튜브 AI가 구두점을 삽입하지 않으므로, 추출된 텍스트에 문장 부호가 포함되지 않습니다. 이는 본 애플리케이션의 한계가 아니라 소스 데이터의 특성입니다.
- **자막 미제공 영상**: 수동 자막과 자동 생성 자막이 모두 비활성화된 영상은 추출이 불가능합니다.
- **언어 설정**: 현재 영어(`en`) 자막만 수집하도록 설정되어 있으며, 코드 내 `subtitleslangs` 값을 변경하면 다른 언어도 지원 가능합니다.
- **번역 품질**: 구두점이 없는 텍스트는 DeepL, Google Translate 등 일반 번역기로 처리할 경우 문장 경계를 인식하지 못해 번역 품질이 저하될 수 있습니다. Claude 등 대형 언어 모델(LLM)에 입력하는 방식이 보다 자연스러운 번역 결과를 제공합니다.
